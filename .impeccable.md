# Impeccable Design Context

## Design Context

### Users
本项目面向知识工作者 / 开发者。用户在桌面环境中高频记录技术片段、项目备忘、命令、链接、排查结论和临时想法，并希望稍后按分类整理与检索。界面应支持快速输入、快速切换分类、快速找到已有内容，而不是偏营销展示或低密度消费型笔记。

### Use Cases
- 快速暂存：临时记录命令、链接、想法、排查记录。
- 分类整理：把便签归入项目、代码片段、临时收集等大分类。
- 日后检索：在当前分类内搜索便签正文，快速找回技术内容。
- 本地工作台：数据主要本地持久化，用户需要明确的保存状态、未保存保护和误删恢复路径。

### Brand Personality
开发者工具感、安静专注、内容优先。

界面应感觉像一个可靠的本地笔记工作台：紧凑、有秩序、低噪声、响应快。避免太可爱、太营销、太玻璃拟态、太霓虹。视觉重点应落在内容、分类、搜索和保存状态上。

### Aesthetic Direction
采用“安静的开发者工作台”方向：深色但不发光，层级克制，卡片边界清晰，操作贴近内容。布局以左侧分类栏 + 右侧便签工作区为主。桌面端保持高效率密度；移动端改为分类折叠/抽屉 + 单列便签。

### Design Principles
1. 内容优先：便签正文、当前分类和搜索结果比装饰元素更重要。
2. 操作归属清晰：分类操作留在侧栏，便签操作留在当前分类内容区。
3. 不隐藏核心路径：新建、搜索、保存、删除、编辑必须可发现、可键盘操作、可触控。
4. 密度服务效率：桌面端允许紧凑，但不能牺牲可读性、焦点状态和 44px 触控命中区。
5. 状态必须可见：未保存、已保存、保存失败、搜索无结果、删除撤销都要有明确反馈。
6. 统一 token：页面、Element Plus、TinyMCE 内容区使用同一套语义颜色、间距和状态 token。
